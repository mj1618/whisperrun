# Task: Runner Gameplay (Milestone 3)

## Overview

Build the complete Runner gameplay experience. When the game starts, the Runner player should see a top-down view of the apartment, centered on their character, with limited visibility (fog of war). They can move with WASD, crouch with Shift, hide in hide spots, pick up the target item, and escape through the exit to win. The Whisper player will see the same `GameCanvas` for now (Whisper-specific view is Milestone 4), but the Runner's gameplay must be fully functional.

This is the first milestone where the game becomes *playable* — a player can actually move around, interact with objects, and achieve a win condition.

## What to Build

### 1. Convex Game Mutations (`/convex/game.ts` — NEW FILE)

Create server-side mutations for Runner actions:

#### `getGameState` (query)
- Args: `{ roomId: v.id("rooms") }`
- Returns the `gameState` document for the given room
- This is what clients subscribe to for real-time game state

#### `moveRunner` (mutation)
- Args: `{ roomId: v.id("rooms"), x: v.number(), y: v.number(), crouching: v.boolean() }`
- Validates position is walkable (you'll need to import/replicate the `isWalkable` check from the map — since Convex runs server-side, you can't import from `src/`. Inline the map data or create a shared map lookup. **Simplest approach:** just trust the client position for now and add server validation later in Milestone 6. Store x,y as tile coordinates (can be fractional for smooth movement).)
- Updates `runner.x`, `runner.y`, `runner.crouching` in gameState

#### `interactRunner` (mutation)
- Args: `{ roomId: v.id("rooms"), sessionId: v.string(), action: v.union(v.literal("hide"), v.literal("unhide"), v.literal("pickup"), v.literal("exit")) }`
- **"hide"**: Set `runner.hiding = true` (client should only send this when near a HideSpot tile)
- **"unhide"**: Set `runner.hiding = false`
- **"pickup"**: Find the first non-picked-up item within 1 tile of runner position. Set that item's `pickedUp = true` and `runner.hasItem = true`. If no item nearby, ignore.
- **"exit"**: If runner has the item AND is within 1 tile of `(exitX, exitY)`, set `phase = "escaped"`. This is the win condition!

#### `startHeistPhase` (mutation)
- Args: `{ roomId: v.id("rooms") }`
- Transitions `phase` from `"planning"` to `"heist"`
- The planning phase gives the Whisper time to look at the map before the Runner moves (for now, either player can trigger this, or auto-transition after a few seconds — keep it simple: just have a "Start!" button)

### 2. Game State Manager (`/src/game/game-state.ts` — NEW FILE)

A bridge between Convex subscriptions and the canvas game loop. The game loop needs synchronous access to the latest game state each frame.

```typescript
export interface LocalGameState {
  runner: { x: number; y: number; crouching: boolean; hiding: boolean; hasItem: boolean };
  guards: Array<{ id: string; x: number; y: number; angle: number; state: string }>;
  items: Array<{ id: string; x: number; y: number; pickedUp: boolean; name: string }>;
  pings: Array<{ x: number; y: number; type: string; createdAt: number }>;
  exitX: number;
  exitY: number;
  phase: string;
  startTime: number;
}
```

- Store latest game state from Convex subscription
- Provide `getState(): LocalGameState | null` for the game loop to read each frame
- For the Runner's local movement: maintain a local runner position that gets rendered immediately (client prediction), while also sending updates to Convex. The Convex subscription will confirm/correct position. Use a simple approach: render local position, send position to server at a throttled rate (~10 updates/sec), and snap to server position when it arrives if the difference is small.

### 4. Fog of War (`/src/game/runner-view.ts` — NEW FILE)

Implement limited visibility for the Runner.

**Approach:** After rendering the full tile map, overlay a fog layer that blacks out everything outside the Runner's visibility radius.

- Visibility radius: 6 tiles (192 pixels) when walking, 4 tiles (128 pixels) when crouching
- Use a radial gradient: fully transparent at center, fully opaque black at the edge of the radius
- Implementation: draw the tile map normally, then draw a full-canvas dark overlay, then "cut out" a circle at the Runner's position using `globalCompositeOperation = 'destination-out'` on an offscreen canvas, then composite the result.
- **Simpler alternative (use this one):** Create a full-canvas overlay and use `ctx.arc()` with a radial gradient clip to create the visibility hole. Specifically:
  1. Save context state
  2. Create a path with `ctx.rect(0, 0, width, height)` then `ctx.arc(runnerScreenX, runnerScreenY, radius, 0, Math.PI*2, true)` (the `true` makes it counter-clockwise, creating a hole)
  3. Clip to this path
  4. Fill with dark color (rgba(0,0,0,0.85))
  5. Restore context state
- Add a soft edge using a radial gradient drawn at the Runner's position on top

Export a function: `renderFogOfWar(ctx, canvasWidth, canvasHeight, runnerScreenX, runnerScreenY, radius)`

## Key Technical Details

### Movement & Collision
- Runner position stored as fractional tile coordinates (e.g., `x: 3.5, y: 7.2` means halfway through tile col 3, near the top of tile row 7)
- Movement speed: 3 tiles/sec = 96 px/sec; crouching: 1.5 tiles/sec = 48 px/sec
- Collision: before moving, check the tiles the Runner would overlap. The Runner hitbox is ~0.6 tiles wide (centered on position). Check all four corners of the hitbox against `isWalkable()`.
- Axis-separated collision: try X movement first, then Y. This prevents getting stuck on corners ("wall sliding").

### Convex Integration in Game Loop
- React hooks (`useQuery`, `useMutation`) run in React-land. The game loop runs in requestAnimationFrame.
- **Bridge pattern:** Use `useRef` to store the latest game state and mutation functions. The game loop reads from these refs each frame.
- For position updates: accumulate movement locally, send to Convex on a throttled interval (every 100ms). Don't `await` the mutation in the game loop — fire and forget.
- Game state subscription: when new state arrives from Convex, update the ref. The next frame will use it.

### Interaction Detection
- When player presses Space or E:
  - Check if Runner is within 1.5 tiles of a HideSpot tile → toggle hide
  - Check if Runner is within 1.5 tiles of a non-picked-up item → pickup
  - Check if Runner is within 1.5 tiles of exit AND has item → trigger exit/win
- Priority: exit > pickup > hide (so you don't accidentally hide when trying to escape)
