# Task: Visual Heist Replay Map on Results Screen

## Overview

The results screen currently shows a text-only "Highlight Reel" — a list of narrative strings like "A guard spotted you!" with timestamps. This is functional but misses the game's biggest opportunity for shareability and engagement: **seeing your heist path visually**.

This task adds a **Visual Heist Replay Map** to the results screen: a small canvas that renders the building blueprint with the Runner's path drawn as an animated line, key event markers (guard alerts, near misses, item pickup, exit/caught location), and a playback timeline. Think of it like the "route replay" in running apps, or the post-match heatmap in competitive games — it transforms the text highlight reel into something visual, memorable, and shareable.

This is the single highest-impact feature for the "clip-able co-op moments" goal in the design doc. It builds directly on the existing `EventRecorder`, `generateMap()`, and `generateHighlights()` infrastructure with no new backend changes.

## What to Build

### 1. Add Position Tracking to EventRecorder (`/src/game/events.ts` — MODIFY)

The current event recorder captures discrete events (guard alerts, near misses, etc.) but does NOT track the Runner's position over time. For the replay map, we need a lightweight position trail.

Add a new event type `"position"` and a dedicated position trail array for efficiency:

```typescript
// Add to GameEventType union:
| "position"

// Add to GameEvent data:
data?: {
  // ... existing fields ...
  crouching?: boolean;  // NEW: track crouching state at position snapshots
};

// Add to EventRecorder class:
private positionTrail: Array<{ x: number; y: number; t: number; crouching: boolean }> = [];

recordPosition(x: number, y: number, crouching: boolean): void {
  const t = Date.now() - this.heistStartTime;
  // Only record if position changed meaningfully (> 0.3 tiles) — keeps trail compact
  const last = this.positionTrail[this.positionTrail.length - 1];
  if (last) {
    const dx = x - last.x;
    const dy = y - last.y;
    if (dx * dx + dy * dy < 0.09) return; // 0.3^2 = 0.09
  }
  this.positionTrail.push({ x, y, t, crouching });
}

getPositionTrail(): Array<{ x: number; y: number; t: number; crouching: boolean }> {
  return [...this.positionTrail];
}
```

Also update `start()` to clear the trail, and `reset()` to clear it too.

**Important:** `recordPosition` should be called from the game loop at the same place Runner position is updated — roughly every frame during the heist phase. The 0.3-tile threshold ensures we don't store thousands of identical points while still capturing smooth movement.

### 4. Create the ReplayMap Component (`/src/components/ReplayMap.tsx` — CREATE)

This is the core of the feature. Create a new component that renders a small canvas showing:
- The building layout (walls, floors, doors) — rendered from the regenerated map
- The Runner's path as an animated line

- A playback timeline

"use client";

**Rendering approach:**

The canvas should be sized to fit the results screen card — roughly 360px wide, with height proportional to the map aspect ratio. Use a scale factor: `canvasWidth / (map.width * TILE_SIZE)`.

**Map rendering (static background):**
- Render once to an offscreen canvas and cache it
- Walls: dark gray (`#1a1a2e`)
- Floors: slightly lighter (`#16213e` or `#1e2d4a`)
- Doors: thin line in a different shade
- Items/exit/spawn: small colored dots (gold for item, green for exit, blue for spawn)
- Keep it blueprint-style — minimal, clean, dark background matching the Whisper view aesthetic

**Path rendering (animated):**
- Draw the Runner's trail as a glowing line
- Walking sections: solid gold line (`#FFD700`)
- Crouching sections: dashed, dimmer line (`#FFD700` at 50% opacity)
- Animate by interpolating along the trail over a 5-second playback duration
- Use `requestAnimationFrame` for smooth animation
- The line should appear to "draw itself" from start to end

**Event markers:**
- Show markers at positions where key events occurred
- Guard alert: red pulse circle (⚠)
- Near miss: orange circle
- Item pickup: gold star
- Caught location: red X
- Exit/escape: green checkmark
- Markers appear when the animated path reaches that timestamp
- Keep markers as simple colored circles (4-6px radius) with a brief scale-in animation

**Timeline bar:**
- Thin horizontal bar below the canvas showing elapsed time
- Fill bar animates alongside the path drawing
- Show total heist duration on the right

**Playback controls:**
- Auto-plays on mount (after a 0.5s delay)
- "Replay" button to restart the animation
- No pause/scrub needed — keep it simple

**Implementation notes:**
- Use `useMemo` to regenerate the map from seed + difficulty (same as GameCanvas does)
- Render the static map to an offscreen `OffscreenCanvas` or a second canvas element, then `drawImage` it each frame
- The animation loop should use `requestAnimationFrame` and cancel on unmount
- Handle edge case: no position trail (Whisper role, or old events without position data) → show static map with event markers only, no animated path

## Key Design Decisions (Already Made)

1. **Client-side only** — No backend changes. The position trail and events are already stored client-side in the EventRecorder. The map is regenerated from the seed. This is the cleanest approach with zero backend cost.
2. **0.3-tile threshold for position recording** — Records position only when the Runner moves meaningfully. A 3-minute heist with constant movement yields ~600-1800 points, well within memory budget. A crouching player produces fewer points.
3. **5-second animated playback** — Maps the entire heist duration to a 5-second animation. Fast enough to watch, slow enough to follow the path. No need for real-time playback (a 3-minute replay would be boring).
4. **Blueprint aesthetic** — The replay map uses the Whisper's dark blueprint color scheme, not the Runner's warm in-game palette. This creates visual continuity with the "overview" perspective and looks cleaner at small sizes.
5. **No new backend tables** — Everything needed for the replay is already available client-side (events + map seed + difficulty). The position trail is ephemeral — it exists for the current session only and doesn't need persistence.
6. **Graceful degradation** — If no position trail is available (Whisper role, or old clients), the replay section simply doesn't render. No errors, no empty state.
7. **Auto-play with replay button** — The animation auto-starts for immediate gratification. A small "Replay" button allows re-watching. No complex controls (pause, scrub, speed) — keep it delightful and simple.

Fixed by guarding the denominator: `const dt = pt.t - prev.t; const segFraction = dt > 0 ? (currentHeistTime - prev.t) / dt : 0;`. This edge case can occur if the runner records two positions at the same millisecond.
